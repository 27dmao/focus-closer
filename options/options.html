    Shortcuts

    ⌘+Shift+XMark distracting — closes tab + auto-blocks the channel
    ⌘+Shift+SMark productive — whitelists current video + channel, OR undoes the last close (within 5 min)
    ESCDismiss the close popup
    The two feedback shortcuts (X and S) are how the system learns. Each press updates the channel whitelist/blocklist, so all future videos from that channel are handled correctly without ever calling Claude. The classifier converges fast. (On Windows/Linux: Ctrl instead of ⌘.)
    Rebind at chrome://extensions/shortcuts.

    Classification waterfall

    1. User override — flagged videos always close; reopened videos always stay.
    2. Cache — 30-day TTL. Bypassed during Focus Sessions.
    3. Local rules — channels + title keywords. Covers ~50% of traffic.
    4. Claude Haiku 4.5 — the rest. ~$0.0005/call, ~500ms, prompt-cached.
    5. Fail-open — Claude down → tab stays.
    Eval harness at evals/run.js. Run npm run eval to validate prompt changes against 30 labeled cases.

    Commitment device

    Guest mode, Incognito, and fresh Chrome profiles bypass any extension. Pair this with a managed Chrome policy (on macOS: /Library/Managed Preferences/com.google.Chrome.plist, requires sudo). See the README.
